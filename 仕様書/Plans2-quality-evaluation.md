# SEO記事生成システム品質評価計画

> **作成日**: 2026-02-03
> **目的**: 手動生成の良記事とシステム生成記事を比較し、品質差異を定量・定性評価する

---

## 概要

### 目標

1. 現在のシステムで4つのキーワードに対して記事を生成
2. `/tmp/blog-articles/` の良記事（手動生成）と比較
3. 品質差異を評価・レポート化

## 🟢 フェーズ3: 品質評価 `cc:TODO`

### 3.1 定量評価

| 評価項目 | 評価方法 |
|---------|---------|
| 文字数 | 元記事 vs 生成記事の文字数比較 |
| 見出し数 | H2/H3の数と構造 |
| CTA配置 | 指定位置への配置確認 |
| キーワード密度 | メインKW出現回数/総文字数 |
| 工程処理時間 | 各工程の実行時間 |

### 3.2 定性評価（Codex レビュー）

| 評価項目 | 評価基準 |
|---------|---------|
| 導入部の訴求力 | 読者の課題への共感、興味喚起 |
| 構成の論理性 | 見出し間の流れ、網羅性 |
| 具体性 | データ/事例の活用 |
| CTA誘導 | 自然な導線設計 |
| SEO適合性 | メインKW・関連KWの自然な配置 |

## 実行コマンド

### Run作成（API呼び出し例）

```bash
curl -X POST http://localhost:8000/api/runs \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d '{
    "input": {
      "format_type": "legacy",
      "keyword": "ドライバー 採用 難しい",
      "target_audience": "ドライバーの人材獲得に悩んでいる採用担当者"
    },
    "model_config": {
      "platform": "gemini",
      "model": "gemini-2.0-flash",
      "options": {}
    }
  }'
```

### Run状態確認

```bash
curl http://localhost:8000/api/runs/<run_id> \
  -H "Authorization: Bearer <TOKEN>"
```
