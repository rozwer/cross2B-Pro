## 基本方針

- 適材適所: 各エージェントの専門領域に応じて使い分ける
- 親への報告: 重大な判断・エラーは親エージェントに戻す

## エージェント選択基準

| 場面 | エージェント |
|------|-------------|
| 設計判断 | @architect |
| コミット作成 | @commit-creator |
| PR 作成 | @pr-creator |
| エラー分析 | @error-analyzer |
| リファクタ | @refactorer |

## 親に戻すべき状況

- ロジック変更を伴う修正が必要
- 複数ファイルに跨る大規模リファクタ
- 仕様変更の判断が必要
- セキュリティリスクを発見
