# /review - コードレビュー

## 概要

指定されたコードや変更内容をレビューし、問題点・改善案を報告する。

## 使用例

- `/review` - 未コミット変更をレビュー
- `/review <file>` - 特定ファイルをレビュー
- `/review --pr=<number>` - PR をレビュー

## レビュー観点

1. **バグ**: 潜在的なバグ、エッジケース
2. **セキュリティ**: 脆弱性、機密情報の扱い
3. **パフォーマンス**: N+1、不要なループ
4. **可読性**: 命名、コメント、構造
5. **テスト**: カバレッジ、テストケースの網羅性

## 出力形式

```markdown
## High Priority
- [問題内容]
  - 場所: file:line
  - 修正案: ...

## Medium Priority
- [問題内容]
  - 場所: file:line
  - 修正案: ...

## Low Priority / Suggestions
- [提案内容]
```

## 使用するエージェント

- @codex-reviewer: Codex 経由のレビュー（設定時）
