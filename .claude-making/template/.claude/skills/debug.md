# /debug - デバッグ支援

## 概要

発生しているエラーや問題を調査し、原因と解決策を特定する。

## 使用例

- `/debug <エラーメッセージ>`
- `/debug --logs` - ログを分析

## 実行フロー

1. エラー情報を収集
2. スタックトレースを分析
3. 関連コードを調査
4. 原因を特定
5. 解決策を提示

## デバッグ手法

- ログ分析
- スタックトレース解析
- 変数状態の確認
- 再現手順の特定

## 使用するエージェント

- @error-analyzer: エラー分析
- @stack-tracer: スタックトレース解析
- @log-investigator: ログ調査
