# /push - プッシュ

## 概要

現在のブランチをリモートリポジトリにプッシュする。事前にコミット漏れや未追跡ファイルをチェック。

## 使用例

- `/push` - 現在のブランチをプッシュ
- `/push --force` - 強制プッシュ（要注意）

## 実行フロー

1. `git status` で未コミット変更を確認
2. 未コミット変更があれば警告
3. リモートとの差分を確認
4. `git push` を実行
5. 結果を報告

## 注意事項

- main/master への直接プッシュは禁止（PR 経由を推奨）
- --force は慎重に使用すること

## 使用するエージェント

- @push-handler: プッシュ前チェックと実行
