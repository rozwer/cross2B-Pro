# WordPressブロック対応の基本原則

## 1. Gutenbergが認識するHTML構造

WordPressのブロックエディタは、特定のHTML構造をネイティブブロックとして認識します。

### 見出しブロック（Heading）
```html
<h2>見出しテキスト</h2>
<h3>小見出しテキスト</h3>

```

### 段落ブロック（Paragraph）
```html
<p>段落テキスト。複数の文を含めることができます。</p>

```

### リストブロック（List）
```html
<ul>
<li>リスト項目1</li>
<li>リスト項目2</li>
<li>リスト項目3</li>
</ul>

```

番号付きリスト:
```html
<ol>
<li>手順1</li>
<li>手順2</li>
<li>手順3</li>
</ol>

```

### 引用ブロック（Quote）
```html
<blockquote>
<p>引用テキスト</p>
<cite>引用元</cite>
</blockquote>

```

### 画像ブロック（Image）- プレースホルダー
```html
<figure class="wp-block-image">
<img src="[画像URL]" alt="[代替テキスト]">
<figcaption>[キャプション]</figcaption>
</figure>

```

### テーブルブロック（Table）
```html
<figure class="wp-block-table">
<table>
<thead>
<tr>
<th>ヘッダー1</th>
<th>ヘッダー2</th>
</tr>
</thead>
<tbody>
<tr>
<td>データ1</td>
<td>データ2</td>
</tr>
</tbody>
</table>
</figure>

```

### 区切りブロック（Separator）
```html
<hr class="wp-block-separator">

```

### ボタンブロック（Buttons）- CTA用
```html
<div class="wp-block-buttons">
<div class="wp-block-button">
<a class="wp-block-button__link" href="[URL]">[ボタンテキスト]</a>
</div>
</div>

```

---

## 2. 変換ルール（Markdown → WordPress HTML）

### 見出しの変換

| Markdown | WordPress HTML |
|----------|----------------|
| `# タイトル` | `<h1>タイトル</h1>` |
| `## 大見出し` | `<h2>大見出し</h2>` |
| `### 小見出し` | `<h3>小見出し</h3>` |

### 段落の変換

| Markdown | WordPress HTML |
|----------|----------------|
| 通常のテキスト | `<p>テキスト</p>` |
| 空行で区切られた段落 | 別々の`<p>`タグ |

### 強調の変換

| Markdown | WordPress HTML |
|----------|----------------|
| `**太字**` | `<strong>太字</strong>` |
| `*斜体*` | `<em>斜体</em>` |

### リストの変換

| Markdown | WordPress HTML |
|----------|----------------|
| `- 項目` | `<ul><li>項目</li></ul>` |
| `1. 項目` | `<ol><li>項目</li></ol>` |

### リンクの変換

| Markdown | WordPress HTML |
|----------|----------------|
| `[テキスト](URL)` | `<a href="URL">テキスト</a>` |

### 引用の変換

| Markdown | WordPress HTML |
|----------|----------------|
| `> 引用文` | `<blockquote><p>引用文</p></blockquote>` |

## 3. CTA（コール・トゥ・アクション）の変換

CTAはボタンブロックとして変換します。

### Early CTA / Mid CTA / Final CTA

```html
<!-- CTA: [CTA名] -->
<div class="wp-block-buttons is-layout-flex">
<div class="wp-block-button">
<a class="wp-block-button__link wp-element-button" href="[CTAのURL]">[CTAのテキスト]</a>
</div>
</div>

```

**注意**:
- CTAのURLとテキストは工程0で定義されたものを使用
- ボタンのスタイルはWordPress側で調整可能

## 4. 画像プレースホルダーの配置

工程11の画像提案に基づき、画像挿入位置にプレースホルダーを配置します。

### 画像プレースホルダーの形式

```html
<!-- 画像挿入位置: [画像番号] [画像タイトル] -->
<figure class="wp-block-image size-large">
<img src="#IMAGE_PLACEHOLDER_[番号]" alt="[alt属性テキスト]">
<figcaption>[画像の説明・キャプション]</figcaption>
</figure>
<!-- /画像挿入位置 -->

```

**例**:
```html
<!-- 画像挿入位置: 1 派遣社員教育の5ステップフローチャート -->
<figure class="wp-block-image size-large">
<img src="#IMAGE_PLACEHOLDER_1" alt="派遣社員教育の5ステップを示すフローチャート">
<figcaption>派遣社員教育の5ステップ</figcaption>
</figure>
<!-- /画像挿入位置 -->

```

### アイキャッチ画像

アイキャッチ画像は記事本文には含めず、別途指示として記載します。

```html
<!--
=============================================
アイキャッチ画像の設定
=============================================
画像ファイル: [推奨ファイル名]
alt属性: [alt属性テキスト]
※WordPressの「アイキャッチ画像」機能で設定してください
=============================================
-->

```

## 5. FAQ構造化データ（Schema.org）

FAQセクションにはSchema.org構造化データを追加し、検索結果でのリッチスニペット表示を狙います。

### FAQ HTML構造

```html
<div itemscope itemtype="https://schema.org/FAQPage">

<h2>よくある質問（FAQ）</h2>

<div itemscope itemprop="mainEntity" itemtype="https://schema.org/Question">
<h3 itemprop="name">Q1. 質問文</h3>
<div itemscope itemprop="acceptedAnswer" itemtype="https://schema.org/Answer">
<p itemprop="text">回答文</p>
</div>
</div>

<div itemscope itemprop="mainEntity" itemtype="https://schema.org/Question">
<h3 itemprop="name">Q2. 質問文</h3>
<div itemscope itemprop="acceptedAnswer" itemtype="https://schema.org/Answer">
<p itemprop="text">回答文</p>
</div>
</div>

</div>

```

## 6. 目次の自動生成対応

WordPressの目次プラグイン（Table of Contents Plus、Easy Table of Contents等）が自動で目次を生成できるよう、見出しにはIDを付与します。

```html
<h2 id="section-1">見出し1</h2>
<h3 id="section-1-1">小見出し1-1</h3>
<h2 id="section-2">見出し2</h2>

```

**ID命名規則**:
- H2: `section-[番号]`（例: section-1, section-2）
- H3: `section-[H2番号]-[H3番号]`（例: section-1-1, section-1-2）

# 注意事項

## WordPressへの貼り付け方法

1. **カスタムHTMLブロック**として貼り付け
2. **「ブロックへ変換」**を実行
3. 各要素が独立したブロックになる
4. 画像プレースホルダーを実際の画像に置き換え

## 編集しやすさを優先

- 複雑なネストは避ける
- 各ブロックが独立して編集できる構造
- クラス名はWordPress標準のものを使用

## SEO対応

- 見出しにID付与（目次プラグイン対応）
- FAQ構造化データ（リッチスニペット対応）
- 画像のalt属性を必ず設定

## 画像について

- プレースホルダーは`#IMAGE_PLACEHOLDER_[番号]`形式
- 実際の画像URLは手動で置き換え
- WordPress上でメディアライブラリから選択も可能
