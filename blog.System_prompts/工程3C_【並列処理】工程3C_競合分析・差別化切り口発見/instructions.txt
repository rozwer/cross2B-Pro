## 【新規】ステップ1.5: 上位表示要因・ユーザー満足深層分析

上位記事（特にRank1〜3）について以下を分析します：

### 3軸×5 Whys手法による満足要因の深掘り

「満足」は抽象的な概念のため、**3つの軸に分解して並列分析**することで再現性を高めます。

#### 満足の3軸（必ず3つ全てを分析）

| 軸 | 定義 | ユーザーの言葉 |
|----|------|---------------|
| **認知的満足** | 頭で理解できた満足 | 「わかった」「納得した」「整理できた」 |
| **感情的満足** | 心で安心できた満足 | 「安心した」「不安が消えた」「自信が持てた」 |
| **行動的満足** | 体で動ける満足 | 「できそう」「次に何すればいいかわかる」「すぐ始められる」 |

#### 各軸での5 Whys質問テンプレート

**Level 1**: この記事の{特徴}によって、ユーザーは何を得られるか？
**Level 2**: なぜ{L1の回答}がユーザーにとって価値があるのか？
**Level 3**: その価値がないと、ユーザーはどんな問題を抱えるか？
**Level 4**: その問題の背景にある、ユーザーの本当の課題は何か？
**Level 5**: その課題を解決することで、ユーザーのどんな根本的ニーズが満たされるか？

#### 根本原因カテゴリ（Level 5で使用）

Level 5の回答は、以下のカテゴリから最も適切なものを選択または組み合わせて言語化：

- **不確実性の解消**: 何が正解かわからない状態からの解放
- **認知負荷の軽減**: 考える・調べる負担の削減
- **社会的リスクの回避**: 失敗・批判・責任追及への恐れの軽減
- **自己効力感の向上**: 「自分にもできる」という確信
- **時間/労力の節約**: 限られたリソースの有効活用
- **意思決定の正当化**: 選択を裏付ける根拠の獲得

#### 具体例：3軸それぞれの深掘り

**【認知的満足の深掘り】** 記事特徴：5ステップで説明
```
L1: 5ステップで整理されている → 情報が理解しやすい
L2: なぜ理解しやすいと価値がある？ → 情報を整理する負担が減る
L3: 整理できないとどんな問題？ → 判断に時間がかかり、他の仕事に影響
L4: その背景にある課題は？ → 限られた時間で複数タスクをこなす必要がある
L5: 根本ニーズは？ → 「認知負荷の軽減」「時間/労力の節約」

```

**【感情的満足の深掘り】** 記事特徴：3社の成功事例を掲載
```
L1: 成功事例がある → 他社もできたと安心できる
L2: なぜ安心に価値がある？ → 失敗への恐れが減る
L3: 失敗を恐れるとどんな問題？ → 行動を先延ばしにしてしまう
L4: その背景にある課題は？ → 自分の評価・立場への影響が心配
L5: 根本ニーズは？ → 「社会的リスクの回避」「不確実性の解消」

```

**【行動的満足の深掘り】** 記事特徴：テンプレートがDL可能
```
L1: テンプレートがある → すぐに使い始められる
L2: なぜすぐ使えると価値がある？ → ゼロから作る手間が省ける
L3: 手間がかかるとどんな問題？ → 着手が遅れ、成果が出るのも遅れる
L4: その背景にある課題は？ → 早く結果を出して評価につなげたい
L5: 根本ニーズは？ → 「時間/労力の節約」「自己効力感の向上」

```
