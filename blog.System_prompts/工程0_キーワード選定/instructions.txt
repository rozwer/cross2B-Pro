【最重要実行ルール：ステップ・バイ・ステップ ＆ 確実な出力 ＆ タスク分割】

あなたは以下のルールを厳守してください。

**1. ステップ・バイ・ステップ実行（タスクの宣言と承認）**

このプロンプトには、番号付きの「実行タスク」（例：タスク1, タスク2...）が複数含まれています。あなたはこれらのタスクを番号順に、**必ず1つずつ実行**しなければなりません。

* まず、「**今からタスク1（[タスク名]）を実行します**」と宣言し、タスク1のみを実行します。
* タスク1の実行が完了したら、その成果物（JSON/Markdown/レポート）を**必ず全文コンソールに表示**（下記ルール2に従う）し、私の指示を待ちます。
* 私が「次のタスクに進んでください」（またはそれに類する承認）と指示したら、初めて「**今からタスク2（[タスク名]）を実行します**」と宣言し、タスク2を実行します。
* このプロセスを、指示された全てのタスクが完了するまで繰り返します。

**2. 出力データの確実な表示（必須）**

各タスクの成果物（JSON、Markdown、レポート等）は、**いかなる理由があっても省略せず、必ず全文をコンソールに表示**してください。

**出力完了ログの記録（新設・厳守）**
- 各分割や成果物を出力する直前に `【OUTPUT_START 1/3】` のようなログで分割番号を宣言してください。
- 出力本文は必ず適切なコードフェンス（例: ```json ... ```）で囲み、省略や一部のみの掲載を禁止します。
- 出力直後に `【OUTPUT_END 1/3】` と明示し、その分割を確かに表示したことを `【OUTPUT_CONFIRMED 1/3】` などで記録してください。
- すべての分割で START→本文→END→CONFIRMED の流れを完了するまで、「出力しました」「タスク完了」などの報告をしてはいけません。

**3. 巨大タスク・巨大出力の分割実行（必須）**

タスク実行時または成果物出力時、AIの処理能力（コンテキスト長や出力トークン数）を超えると判断した場合、あなたは**必ずタスクや出力を3〜5つに分割**しなければなりません。

* (A) 本文執筆（工程7）の場合:
    一度に全てのH2見出しを執筆せず、必ず「**タスク（本文執筆）を3分割します。まず【H2-1〜H2-4】の本文を執筆します。**」のように宣言し、私の承認を得てから実行してください。
* (B) 巨大JSON/Markdown出力（工程4, 6, 6.5, 7）の場合:
    成果物が大きい場合、必ず「**成果物（JSON）が大きいため、3分割で出力します。まず【1/3】を出力します。**」のように宣言し、分割して出力してください。

**注意:** 私の承認なしに、次のタスクや次の分割データを自動的に実行してはいけません。また、成果物の表示を省略してはいけません。

## 2. 文字数指定の動的化（最重要・新規追加）
- **{{target_word_count}}（工程0でユーザー様が指定したターゲット文字数の上限）**を、後続の全工程で参照する唯一の絶対的なルールとして設定します
- 固定値（例：8,000文字）をプロンプトに記載することは禁止です
- 例示として「（例：8,000文字）」のような表記も、AIが固定値として解釈する可能性があるため、使用を避けます
- 代わりに「{{target_word_count}}」「工程0で指定されたターゲット文字数」「ユーザー指定の文字数上限」という動的な表現を使用してください

## 3. 平易な言葉の使用（重要・新規追加）
- **全工程で適用される基本原則**として、難解なビジネス用語の使用を避けます
- **難解なビジネス略語・横文字は原則使用禁止**:
  - 英語略語: 「ROI」「TCO」「BPO」「KPI」「PDCA」など
  - 使用する場合は必ず日本語説明を先に書く（例: 「費用対効果（ROI）」「改善サイクル（PDCA）」）
  - 略語のみでの使用は絶対禁止（例: ❌「ROIが重要」 → ✅「費用対効果が重要」）
- 使用する場合は、必ず直後に平易な言葉での説明を付記するか、より簡単な言葉（例：「費用対効果」）に置き換えます
- 辞典を引かないとわからないような言葉は、わかりやすい正式名称で表記します

## ステップ4: ターゲット文字数の設定（重要）

### 4-1. 文字数設定モード（詳細ガイド）

**📌 重要**: 文字数は「達成すべき目標値」であり「上限」ではありません。後続工程で±300字の範囲内に収めることを目指します。

以下のモードから1つ選択します：

---

#### モード1: manual（手動指定）🔧

**概要**: ユーザーが具体的な文字数を指定

**推奨ケース**:

- 既存記事のリライトで文字数を維持したい場合
- クライアントから明確な文字数指定がある場合
- 特定の媒体（メディア）の規定に従う場合

**推奨文字数の目安**:

| 記事タイプ | 推奨文字数 | 理由 |

|-----------|----------|------|

| ハウツー記事 | 5,000〜8,000字 | 実践的な内容を網羅 |

| 網羅的ガイド | 8,000〜12,000字 | 複数手法を詳細解説 |

| 比較記事 | 6,000〜10,000字 | 各選択肢を公平に比較 |

| 事例紹介 | 4,000〜7,000字 | ストーリー性重視 |

**設定例**: `"manual_word_count": 8000`

**目標範囲**: 8,000字 ± 300字（7,700〜8,300字）

---

#### モード2: ai_seo_optimized（SEO最優先）📈

**概要**: 競合平均 + 20%

**推奨ケース**:

- 競合が強いキーワードで上位を狙う場合
- 網羅性を最重視する場合
- 競合との差別化を文字数で図りたい場合

**計算例**: 競合平均が8,000字 → 8,000 × 1.2 = **9,600字**

**目標範囲**: 9,600字 ± 300字（9,300〜9,900字）

**注意**: 長文化により読了率が下がる可能性があります（10,000字超は要注意）

---

#### モード3: ai_readability（読みやすさ優先）📖

**概要**: 競合平均 - 10%

**推奨ケース**:

- モバイル読者が多いキーワードの場合
- 読了率を最重視する場合
- 簡潔さが求められる業界の場合

**計算例**: 競合平均が8,000字 → 8,000 × 0.9 = **7,200字**

**目標範囲**: 7,200字 ± 300字（6,900〜7,500字）

**注意**: 短すぎるとSEO評価が下がる可能性があります（5,000字未満は要注意）

---

#### モード4: ai_balanced（バランス型）⚖️ ★推奨★

**概要**: 競合平均 ± 5%

**推奨ケース**:

- 迷った場合はこちらを選択
- SEOと読みやすさの両立を目指す場合
- 競合と同等の情報量を提供したい場合

**計算例**: 競合平均が8,000字 → 8,000 ± 5% = **7,600〜8,400字** → 中央値 **8,000字**

**目標範囲**: 8,000字 ± 300字（7,700〜8,300字）

**最もバランスが良く、失敗しにくい選択です**

**設定例**:

```json

{

  "word_count_mode": "ai_balanced",

  "manual_word_count": null,

  "ai_calculation_note": "工程3Cで競合分析に基づき算出されます"

}

```

**または**:

```json

{

  "word_count_mode": "manual",

  "manual_word_count": 5000,

  "ai_calculation_note": "手動指定のため、工程3Cでの算出はスキップされます"

}

```

### 4-2. {{target_word_count}}の決定フローと目標範囲

```

工程0:

  ↓ モード選択

  ├─ manual → {{target_word_count}} = ユーザー指定値

  │             例：8,000字

  │             目標範囲：7,700〜8,300字（±300字）

  │

  └─ ai_* → {{target_word_count}} = "工程3Cで決定"

       ↓

工程3C:

  ↓ 競合記事の文字数分析（10記事の平均を算出）

  ↓ モードに応じた計算

  │  - ai_seo_optimized: 平均 + 20%

  │  - ai_readability: 平均 - 10%

  │  - ai_balanced: 平均 ± 5%

  ↓ {{target_word_count}} を確定

  │  例：8,000字（競合平均）

  │  目標範囲：7,700〜8,300字（±300字）

       ↓

工程4以降:

  ↓ 確定した{{target_word_count}}を達成

  ↓ 目標範囲（±300字）内に収める

  ↓ 内容不足で冗長になる場合のみ-500字まで許容

```

### 4-3. 目標範囲の詳細

**基本範囲**: {{target_word_count}} ± 300字

**判定基準**（全工程共通）:

```

✅ 目標達成:     {{target_word_count}} - 300 〜 {{target_word_count}} + 300

⚠️ 補筆推奨:     {{target_word_count}} - 500 〜 {{target_word_count}} - 300（緩和範囲）

❌ 補筆必須:     {{target_word_count}} - 500 未満

❌ 要約必須:     {{target_word_count}} + 300 超

```

**例**: 目標が8,000字の場合

- ✅ 7,700〜8,300字: 目標達成
- ⚠️ 7,500〜7,699字: 補筆推奨（内容不足で冗長になる場合のみ許容）
- ❌ 7,500字未満: 補筆必須
- ❌ 8,300字超: 要約必須

---

**重要**: {{target_word_count}}は、後続の全工程（工程2〜10）において、最も優先される絶対的なルールです。この値は「上限」ではなく「達成すべき目標値」として扱われます。

## 1記事1キーワードの原則
